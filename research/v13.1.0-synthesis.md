## Key Gaps Identified

### Gap 1: No SSE Real-Time Dashboard (P0)

**Problem:** Current `--status-html` dashboard uses `<meta http-equiv="refresh">`
every 30s. Polling-based, high latency, flashing UX.

**Solution:** Add Server-Sent Events (SSE) to existing WebhookHandler class (line 869):
- `GET /live` — SSE stream pushing `event: iteration` JSON events
- `GET /dashboard` — Live HTML page with JavaScript EventSource
- `_broadcast_to_sse_clients()` — Called after write_ledger() in run_loop()

**Design:** See research-dashboard-v3-sse.md (477 lines detailed design).
Module-level `_sse_clients: list[Queue]`, `_sse_clients_lock: threading.Lock`.
SSE handler creates queue per client, run_loop broadcasts after each iteration.

**Files:** scripts/launch-loop.py (WebhookHandler + new module-level globals)

### Gap 2: No Session Self-Healing / Heartbeat (P1)

**Problem:** When a spawned session crashes mid-iteration (network, OOM, timeout),
the daemon has no awareness of partial progress or hung-but-alive sessions.

**Solution:** Heartbeat-based monitoring:
- `--heartbeat-timeout` CLI flag (default: 0 = disabled)
- Spawned session prompt includes heartbeat instructions (touch file every 30s)
- Monitoring thread checks file mtime every 5s
- On expiry: kill child, retry, optionally resume from checkpoints
- Store `heartbeat_ok` boolean in ledger per iteration

**Design:** See research-session-self-healing.md (654 lines detailed design).
Heartbeat file: `/tmp/infinite-loop-heartbeat-{session_id}.touch`
Content: `{"session_id": "...", "iteration": N, "timestamp": ..., "pid": ...}`

**Files:** scripts/launch-loop.py (new CLI flag, monitoring thread, prompt modifier)

## Design Decisions for Next Iteration

### SSE Implementation Strategy

1. Add module-level globals BEFORE the WebhookHandler class:
   ```python
   _sse_clients: list[queue.Queue] = []
   _sse_clients_lock = threading.Lock()
   ```

2. Add `_handle_sse()` and `_handle_dashboard()` to WebhookHandler

3. Add `_broadcast_to_sse_clients(state)` module-level function

4. Call `_broadcast_to_sse_clients(state)` in run_loop() after write_ledger(state)

5. SSE timeout: 30s heartbeat between events, queue.get(timeout=30)

### Heartbeat Implementation Strategy

1. Add `--heartbeat-timeout` argparse flag (int, default 0 = disabled)

2. Define heartbeat file path in prompt: `/tmp/infinite-loop-heartbeat-{pid}.touch`

3. Add heartbeat monitoring thread that:
   - Starts before _execute_iteration()
   - Joins after execution completes
   - Checks file mtime every 5s
   - On expiry: kills child process, sets error flag

4. Store `heartbeat_ok` in iteration record
