## Feature 1: Session Self-Healing (Heartbeat) ‚Äî P1

### What was added

A heartbeat-based session health monitoring system that detects hung Hermes
sessions mid-iteration and automatically kills + retries them.

### Implementation details

**Constants (heartbeat.py section, after `_ERROR_THRESHOLDS`):**
- `HEARTBEAT_DIR = "/tmp"` ‚Äî where heartbeat files are stored
- `HEARTBEAT_PREFIX = "infinite-loop-heartbeat-"` ‚Äî file prefix
- `HEARTBEAT_INTERVAL = 30` ‚Äî spawned session writes heartbeat every 30s
- `HEARTBEAT_GRACE_FACTOR = 2.0` ‚Äî grace period = timeout √ó 2
- `HEARTBEAT_POLL_INTERVAL = 5` ‚Äî daemon polls every 5s
- `HEARTBEAT_KILL_GRACE = 5` ‚Äî SIGTERM then SIGKILL after 5s

**New functions (inserted between GoalSpec and _execute_iteration):**
- `_heartbeat_path(identifier)` ‚Äî returns `/tmp/infinite-loop-heartbeat-{id}`
- `_read_heartbeat(path)` ‚Äî reads/parses heartbeat JSON file
- `_write_heartbeat_file(path, data)` ‚Äî atomic write via .tmp + rename
- `_heartbeat_age(path)` ‚Äî returns seconds since last heartbeat modification
- `_monitor_heartbeat(file, timeout, start, proc)` ‚Äî blocking loop that polls
  every 5s, returns status dict: "alive"|"expired"|"lost"|"completed"
- `_run_heartbeat_monitor(...)` ‚Äî wraps `_monitor_heartbeat` in a daemon thread
  with a timeout cap
- `_kill_session(proc, session_id)` ‚Äî SIGTERM ‚Üí 5s ‚Üí SIGKILL
- `_cleanup_stale_heartbeats()` ‚Äî removes heartbeat files from previous runs
- `_cleanup_heartbeat_file(path)` ‚Äî removes a single heartbeat file

**Modified functions:**
- `_build_delegation_prompt()` ‚Äî new `heartbeat_interval` param; injects
  heartbeat instructions into the spawned session prompt when > 0
- `spawn_delegation_session()` ‚Äî new `heartbeat_timeout` and `iteration_count`
  params; in subprocess mode, when `heartbeat_timeout > 0`, replaces
  `subprocess.run` with `subprocess.Popen` + heartbeat monitor thread
- `_execute_iteration()` ‚Äî new `heartbeat_timeout` param; passes through
  to all `spawn_delegation_session` calls (both single and multi-worker)
- `run_loop()` ‚Äî new `heartbeat_timeout` param; passes through
- `_log_startup_banner()` ‚Äî new `heartbeat_timeout` param; logs heartbeat
  status when enabled

**Error recovery integration:**
- `"heartbeat"` added to `_ERROR_SEVERITY` (severity 5 ‚Äî highest)
- `"heartbeat"` added to `_ERROR_THRESHOLDS` (mild=3, moderate=5, stop=7)

**CLI:**
- `--heartbeat-timeout N` (default: 0 = disabled) ‚Äî seconds of inactivity
  before a session is considered hung. Grace period is always timeout √ó 2
  (total window = timeout √ó 3).

**Prompt injection in spawned sessions:**
When `--heartbeat-timeout` is set, spawned Hermes sessions receive explicit
instructions to run a Python heartbeat command every `HEARTBEAT_INTERVAL`
seconds. The prompt includes the exact Python one-liner to write a heartbeat
file.

**Files modified:**
- `scripts/launch-loop.py` ‚Äî ~280 lines added across 12 locations

## Feature 2: Preflight Hermes Version Check ‚Äî P2

### What was added

A new `check_hermes_version()` static method on `PreflightChecker` that runs
`hermes --version` and displays the installed version. Added to both the
instance `run_all()` and the static `run_all()` preflight methods.

The check is informational (non-blocking) ‚Äî it always returns `passed=True`
even if the version string can't be parsed, but it displays the Hermes version
in the preflight output for troubleshooting:

```
‚úì hermes version: Hermes Agent v0.17.0
```
