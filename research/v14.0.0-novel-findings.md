# v14.0.0 Novel Research Findings (Iteration #10)

## Summary

Iteration #10 research completed on 2026-06-26. This document captures
findings that were **NOT documented by any prior iteration** — novel bugs,
edge cases, and opportunities identified through deep code path analysis.

## BUG: DOM-Based XSS in Dashboard HTML Template

**Severity: MEDIUM** — Affects anyone visiting the live dashboard URL when
iteration data contains crafted task_type or classification values.

**Location:** `scripts/launch-loop.py`, `_SSE_DASHBOARD_HTML_TPL`:

- **L2026-L2028**: `typeTag` is built as raw HTML string with `iter.task_type`
  interpolated via string concatenation into `innerHTML` at L2040:
  ```javascript
  var cls = iter.task_type === 'error' ? 'tag-err' : 'tag-ok';
  typeTag = '<span class="tag ' + cls + '">' + iter.task_type + '</span>';
  ```
- **L2031-L2035**: Same pattern for `classifyTag` with `iter.classification`:
  ```javascript
  classifyTag = '<span class="tag ' + cCls + '">' + iter.classification + '</span>';
  ```
- **L2040**: Both tags injected via `tr.innerHTML = '...' + typeTag + '...' + classifyTag + '...'`

**Impact:** A spawned session could return JSON with `task_type` containing
`</span><script>alert('xss')</script><span>` — this would execute in the
dashboard viewer's browser. The `iter.task_type` and `iter.classification`
values come from spawned Hermes sessions and are NOT sanitized.

**Note:** `iter.summary` is partially sanitized (L2038: `safeSummary = summary.replace(/"/g, '&quot;')`)
but is also injected via innerHTML — the `&quot;` escape only prevents
attribute injection, not tag injection.

**Fix:** Use `textContent` for type tags and classification tags, or
use `createElement` + `textContent` instead of innerHTML string building.

## WARNING: Empty Iterations Broadcast Race

**Severity: LOW** — Edge case when first iteration fails before any record
is appended to the state, and SSE client is connected.

**Location:**
- `_broadcast_to_sse_clients(state)` at L2276
- `_build_sse_payload(state)` at L2296
- Called from `run_loop` at L5960

**Analysis:** In `_build_sse_payload` (L2305):
```python
latest = iterations[-1] if iterations else {}
```

The `if iterations else {}` guard prevents an IndexError. However, if
`_broadcast_to_sse_clients` is called before ANY iteration record is
appended, the payload will contain an empty `{}` for `iteration`, and
the dashboard JS at L2052 checks `data.iteration && data.iteration.n`
before adding a row — so the empty iteration is safely ignored.

**Verdict:** Safe against crash, but produces a useless broadcast. No code
path calls broadcast before the first iteration record is appended, because
broadcast happens at L5960 after `_execute_iteration` (L5397-L5900) completes
which always appends at least an error record.

## WARNING: False Convergence with Empty Summaries

**Severity: LOW** — Triggered only if all spawned sessions return empty
JSON output (no summary field).

**Location:** `_detect_convergence` at L5032-L5079

**Analysis:** At L5050:
```python
is_converged, avg_sim = check_convergence(
    existing_summaries + [combined_summary[:200]],
    threshold=convergence_threshold,
    window=convergence_window,
)
```

If `combined_summary` is empty string `""` and `existing_summaries` contains
only `[""]` entries, `check_convergence` compares identical empty strings
and reports 1.0 similarity — triggering a false convergence stop.

**Risk:** Low in practice because a healthy spawned session always returns
a JSON object with a summary field. Only triggered if:
1. The spawned session crashes silently (outputs nothing)
2. The JSON extraction produces an empty summary
3. Convergence detection is enabled (`--convergence-stop`)

**Fix:** Skip convergence check if `combined_summary` is empty or less than
20 chars (too short for meaningful comparison).

## WARNING: Uncapped summary[:200] in compact mode

**Severity: LOW** — Cosmetic.

**Location:** `_compact_summaries` at L5082

**Analysis:** The condensed summaries at L5105 use `s[:80]` but the
full summary at L5111 uses `combined_summary[:200]`. The 200-char cap is
arbitrary and if a multi-paragraph summary is truncated mid-sentence, the
dashboard displays a cut-off text. Not a bug per se, but an aesthetic issue.

## OK: Sentinel Race Condition Analysis

**Finding:** The sentinel read-and-remove pattern at L2516-L2522 and L5600
creates a TOCTOU window:

```
L5600: check_sentinel(sentinel_path)  → reads + removes file
L5601-L5606: handles "stop" signal
L5609-L5643:  
  if "pause": enters pause loop with NO sentinel file 
```

**Scenario:** User writes "stop" to sentinel → `check_sentinel` reads it and
removes the file → code enters pause loop → sentinel file no longer exists.

**Mitigation found:** The pause loop at L5620 uses `check_sentinel_no_remove()`
which does NOT remove the file. So if user writes "stop" while paused, the
file persists and is detected. The race window is only in the normal
(non-paused) iteration check: between L5600's removal and the code deciding
what to do, a second "stop" write is lost. However, the daemon checks the
sentinel at the START of every iteration, so the user can always write
"stop" again and it will be caught on the next iteration.

**Verdict:** Acceptable risk. The race window is small (~1ms). If the code
enters the pause loop (L5609), the sentinel file is already removed — but
`check_sentinel_no_remove` won't see it. The user can write "stop" again.

## OK: Dashboard Heartbeat Handler

**Finding:** L2103 has an empty heartbeat listener:
```javascript
evtSource.addEventListener('heartbeat', function () {});
```

This is intentional — it prevents the browser EventSource from logging
"unhandled event type" warnings. The heartbeat events carry no data and
exist only to keep the SSE connection alive. The `onerror` handler at
L2108 handles reconnection.
