## New findings — NOT in ENGINEERING_BACKLOG.md

### 1. No type checking in CI (high)
The CI pipeline runs ruff (lint + format) and pytest, but has no mypy/pyright/type-check step. Adding type checking would catch a class of bugs that ruff can't detect (e.g., None-safety, type mismatches).
- File: .github/workflows/ci.yml, Makefile
- Fix: Add mypy target + CI step + pyproject.toml config

### 2. make test re-installs dependencies every run (medium)
The Makefile `test` target runs `pip install -e ".[test]"` before `python -m pytest`. This wastes 2-3 seconds per local test run. The `install-dev` target already installs test deps.
- File: Makefile test target
- Fix: Remove the pip install line from the test target (assume deps are installed)

### 3. No Dependabot/Renovate configuration (medium)
No automated dependency update configuration exists. Dependencies (FastAPI, uvicorn, pytest) won't get automatic PRs for security updates.
- Fix: Add .github/dependabot.yml

### 4. No pi binary availability check in CI (medium)
The CI pipeline doesn't verify that `pi` (the external CLI used by the daemon) is available. A `pi` API change could break the daemon without CI catching it.
- File: .github/workflows/ci.yml
- Fix: Add a smoke-test step that installs `pi` and runs pi-loop --help

### 5. No Ruff config in pyproject.toml (BL-029 tracked)
Confirmed: pyproject.toml lacks [tool.ruff] section. This means ruff uses default rules, which may be too permissive or miss project-specific conventions.

### 6. Test coverage gaps (medium)
Current 156 tests cover: CLI parsing, validation, stats, error_utils, color_utils, config constants. Missing: loop_manager, server endpoints, loop.py execution logic, error_recovery, functions.py, git_utils, heartbeat.
- File: tests/
- Note: BL-002 (zero test coverage) was marked completed but coverage is still partial
