## Implemented Features (v11.13.0)

### Feature 1: YOLO Mode (`--yolo`)
**Status: IMPLEMENTED**

Bypasses all dangerous command approval prompts in spawned sessions. Combine with
`--ignore-rules` for fully autonomous operation where no prompts interrupt execution.

Files modified:
- `scripts/launch-loop.py` — Added `--yolo` argparser flag, passed through to
  `spawn_delegation_session()` and `run_loop()`, forwarded to cmd in subprocess mode.
  Skipped in library mode with log warning.
- `SKILL.md` — Documented in changelog, options table, and pitfalls.
- `scripts/run-loop.sh` — Added flag forwarding.
- `references/cross-iteration-context.md` — Version bump.

### Feature 2: Ignore User Config (`--ignore-user-config`)
**Status: IMPLEMENTED**

Passes `--ignore-user-config` to spawned sessions so they skip `~/.hermes/config.yaml`
and fall back to built-in defaults. Useful for isolated, reproducible runs.

Files modified: Same as Feature 1.

### Feature 3: Source Tagging (`--spawn-source`)
**Status: IMPLEMENTED**

Tags spawned sessions with a custom source label passed as `--source` to `hermes chat -q`.
Default remains empty (uses Hermes' default 'cli'). Set to `infinite-loop` or a custom
value to distinguish loop sessions from manual CLI sessions. Source is stored in the
Hermes session database for filtering.

Files modified: Same as Feature 1.

## Implementation Details

### Changes to `spawn_delegation_session()`:
- 3 new parameters: `yolo`, `ignore_user_config`, `spawn_source`
- Subprocess mode: `--yolo`, `--ignore-user-config`, `--source SPAWN_SOURCE` appended to cmd
- Library mode: silent skip (no AIAgent equivalent for any of these flags)

### Changes to `run_loop()`:
- 3 new parameters threaded through
- Passed to both worker-path and single-execution-path spawn calls

### Changes to `main()`:
- 3 argparser entries: `--yolo` (store_true), `--ignore-user-config` (store_true), `--spawn-source` (str)
- Logged in startup banner
- Stored in state dictionary
- Wired into `run_loop()` call
