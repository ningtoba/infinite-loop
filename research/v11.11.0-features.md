### Architecture of `spawn_delegation_session()` (lines 2344-2596)

Three code paths currently:

1. **Worker URL mode** (lines 2415-2476): HTTP POST to worker URL. Stays as-is for library mode — workers are separate processes.

2. **Direct subprocess mode** (lines 2478-2596, default): Runs `hermes chat -q` via `subprocess.run()`:
   - Builds CLI command with flags
   - Sets timeout via `subprocess.run(timeout=...)`
   - Parses JSON from stdout using `extract_json_from_output()`
   - Validates against output schema if provided

3. The prompt is built by `_build_delegation_prompt()` (lines 1983-2341) — a 360-line function.

### `--use-library` Implementation Plan (Refined)

**Add as 4th code path in `spawn_delegation_session()`:**

```python
# --- Library mode (--use-library) — uses AIAgent.run_conversation() in-process ---
if use_library:
    try:
        from run_agent import AIAgent
        agent = AIAgent(
            model=model or None,
            max_iterations=max_turns,
            enabled_toolsets=list(toolsets),  # already a list
            quiet_mode=True,
            ephemeral_system_prompt=prompt,
            skip_memory=True,
            checkpoints_enabled=checkpoints,
            pass_session_id=pass_session_id,
        )
        with concurrent.futures.ThreadPoolExecutor(max_workers=1) as pool:
            future = pool.submit(agent.run_conversation, user_message=prompt)
            try:
                conv_result = future.result(timeout=timeout_seconds)
            except concurrent.futures.TimeoutError:
                future.cancel()
                # fall through to timeout handling
        session_id = conv_result.get("session_id", getattr(agent, "session_id", ""))
        # Extract summary/context from final_response
        final_response = conv_result.get("final_response", "")
        parsed_json = extract_json_from_output(final_response)
        # ... same result-formatting logic as subprocess mode
    except ImportError:
        _log("[WARN] AIAgent not importable, falling back to subprocess mode")
        # fall through to subprocess mode
```

**Important considerations:**
- ThreadPoolExecutor timeout wrapper is mandatory (AIAgent has no wall-clock timeout)
- `pass_session_id=True` tells the spawned AIAgent to include its session ID in the system prompt (the return dict already has `session_id` though)
- Must fall back to subprocess when `--workers > 1` (GIL release benefit of subprocess)
- Must handle `ImportError` gracefully — the daemon may be running in a minimal Python environment

**sys.path consideration:**
The `run_agent` module is installed in the Hermes venv. The daemon runs inside the Hermes environment, so `from run_agent import AIAgent` should work. The fallback to subprocess mode on ImportError provides safety.

**For subprocess mode** (when `--pass-session-id` is passed to the spawned `chat -q`):

### New Changelog Section for v11.11.0

```
**v11.11.0 enhancements — AIAgent library mode, session tracking, checkpoints (iteration #13):**

1. **AIAgent Library Mode (`--use-library`)** — Instead of spawning `hermes chat -q` as a
   subprocess, the daemon can now import `AIAgent` from `run_agent` and run the conversation
   in-process. This eliminates subprocess overhead, simplifies error handling, and provides
   direct access to the result dict (including session_id, token usage, and cost data).
   Falls back to subprocess mode automatically if the library is not importable.

2. **Session Tracking** — `--pass-session-id` passes the `--pass-session-id` flag to spawned
   Hermes sessions. The daemon extracts the `session_id:` line from spawned stdout and stores
   it in the ledger's `spawned_session_id` field. In library mode, the session_id is obtained
   directly from the AIAgent object.

3. **Checkpoints Flag** — `--checkpoints` enables file checkpoints in spawned sessions.
   Passes `--checkpoints` to the spawned `chat -q` command (subprocess mode) or sets
   `checkpoints_enabled=True` (library mode). Auto-enabled when `--git` is set.
```
